## v0.28.0

### 📣 Highlights

- Better app-installation token handling across services, so auth flows are more reliable in day-to-day use.
- Native object-storage artifact support landed across the stack, making artifact workflows cleaner end to end.
- GitHub merge queue events are now supported, which helps teams using newer PR/merge flows.
- Secret and pipeline handling saw several quality-of-life improvements to reduce surprises during runs.
- Build and hook counter safety was improved to lower collision risk under heavier activity.
- Overall, v0.28 is focused on smoothing rough edges and making the platform feel more predictable.

### 💥 Breaking Changes

#### Admins

GitHub merge queue support changes webhook/event expectations. Ensure your SCM app and repo settings include merge queue event handling where needed.
- (server) feat(events)!: support merge queue for GitHub SCM ([#1343](https://github.com/go-vela/server/pull/1343)) - thanks [@ecrupper](https://github.com/ecrupper)!

Build and hook counters now use repo-tied transactions to prevent collisions. If you have custom DB workflows around counters, validate them after upgrade.
- (server) fix(database)!: use transactions tied to repo counters for hook and build to avoid collisions ([#1353](https://github.com/go-vela/server/pull/1353)) - thanks [@ecrupper](https://github.com/ecrupper)!

Native secret scope toggles can disable management operations by scope at runtime. Review platform secret settings to avoid unexpected create/update/delete failures.
- (server) feat!: platform setting for enabling/disabling Vela secrets ([#1384](https://github.com/go-vela/server/pull/1384)) - thanks [@ecrupper](https://github.com/ecrupper)!

MustRead/MustWrite auth now accepts installation tokens with stricter endpoint permission validation. Integrations using install tokens should verify required app permissions.
- (server) enhance(api)!: accept installation tokens for MustWrite and MustRead auth using caching ([#1373](https://github.com/go-vela/server/pull/1373)) - thanks [@ecrupper](https://github.com/ecrupper)!

#### Users

All SDK methods now require a context parameter. Update callers to pass `context.Context` (for example, `context.Background()` or request-scoped contexts).
- (sdk-go) enhance!: add context param to all SDK funcs ([#371](https://github.com/go-vela/sdk-go/pull/371)) - thanks [@ecrupper](https://github.com/ecrupper)!

Origin secrets with the same `name` are now deduplicated. Pipelines that relied on duplicate names or order-dependent collisions should be reviewed.
- (server) enhance(compiler)!: do not add duplicate origin secrets using `name` as key ([#1390](https://github.com/go-vela/server/pull/1390)) - thanks [@ecrupper](https://github.com/ecrupper)!

Buildkite warning-only behavior has been removed. If your pipelines previously relied on those warnings (without failing), they may now fail and require updates to pass under the new behavior.
- (server) feat: remove buildkite dependency ([#1352](https://github.com/go-vela/server/pull/1352)) - thanks [@ecrupper](https://github.com/ecrupper)!
